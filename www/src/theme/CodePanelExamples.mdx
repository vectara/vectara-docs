# CodePanel Examples

This page demonstrates all CodePanel features with 10 examples, from simplest to most complex.

## Example 9: Complex Multi-Feature Example

Combining multiple features: tabs, highlighting, annotations, and stacked layout.

<CodePanel
  title="Complete Integration Guide"
  layout="stacked"
  tabs={true}
  highlightLines="5-7,15-17"
  annotations={{
    javascript: [
      { line: 5, text: "Initialize with your credentials" },
      { line: 15, text: "Handle the search results" }
    ],
    python: [
      { line: 5, text: "Create client instance" },
      { line: 15, text: "Process search results" }
    ]
  }}
  snippets={[
    {
      language: 'javascript',
      title: 'Node.js',
      code: `// Vectara Search Integration
const { VectaraClient } = require('@vectara/sdk');

// Setup client
const client = new VectaraClient({
  apiKey: process.env.VECTARA_API_KEY,
  customerId: process.env.VECTARA_CUSTOMER_ID
});

// Perform search
async function search(query) {
  try {
    const results = await client.search({
      query: query,
      numResults: 10,
      corpusId: 1
    });
    
    return results.documents;
  } catch (error) {
    console.error('Search failed:', error);
    throw error;
  }
}`
    },
    {
      language: 'python',
      title: 'Python',
      code: `# Vectara Search Integration
from vectara import Client, SearchRequest
import os

# Setup client
client = Client(
    api_key=os.environ['VECTARA_API_KEY'],
    customer_id=os.environ['VECTARA_CUSTOMER_ID']
)

# Perform search
def search(query: str):
    try:
        request = SearchRequest(
            query=query,
            num_results=10,
            corpus_id=1
        )
        
        results = client.search(request)
        return results.documents
    except Exception as error:
        print(f"Search failed: {error}")
        raise`
    }
  ]}
/>

## Example 10: Advanced Editable Example with Custom Runner

The most complex example: editable code with custom execution handler for server-side processing.

<CodePanel
  title="Interactive Query Builder"
  editable={true}
  tabs={true}
  layout="stacked"
  defaultOutput="Modify the query and click Reset to execute"
  onRun={async (code, language) => {
    // Custom execution logic here
    if (language === 'json') {
      try {
        const query = JSON.parse(code);
        return `Query validated successfully!\n${JSON.stringify(query, null, 2)}`;
      } catch (e) {
        return `JSON Error: ${e.message}`;
      }
    }
    return `Language ${language} execution not implemented`;
  }}
  snippets={[
    {
      language: 'json',
      title: 'Query JSON',
      code: `{
  "query": "What is Vectara?",
  "search_params": {
    "corpus_ids": [1, 2, 3],
    "num_results": 10,
    "rerank": true,
    "reranker": {
      "type": "mmr",
      "diversity_factor": 0.3
    }
  },
  "metadata_filter": {
    "category": {
      "operator": "in",
      "values": ["tutorial", "guide", "reference"]
    },
    "date": {
      "operator": "gte",
      "value": "2024-01-01"
    }
  },
  "summary": {
    "enabled": true,
    "max_length": 200,
    "response_lang": "en"
  }
}`
    },
    {
      language: 'javascript',
      title: 'Query Builder',
      code: `// Build a Vectara query programmatically
const query = {
  query: "What is Vectara?",
  searchParams: {
    corpusIds: [1, 2, 3],
    numResults: 10,
    rerank: true,
    reranker: {
      type: "mmr",
      diversityFactor: 0.3
    }
  },
  metadataFilter: {
    category: {
      operator: "in",
      values: ["tutorial", "guide", "reference"]
    },
    date: {
      operator: "gte",
      value: "2024-01-01"
    }
  },
  summary: {
    enabled: true,
    maxLength: 200,
    responseLang: "en"
  }
};

// Convert to JSON
console.log("Generated Query:");
console.log(JSON.stringify(query, null, 2));`
    }
  ]}
/>

## Summary

These examples demonstrate the full range of CodePanel capabilities:

1. **Basic** - Simple code display
2. **Multi-language** - Language selector dropdown
3. **Layout** - Stacked layout with custom width
4. **Highlighting** - Line highlighting for emphasis
5. **Tabs** - Tab-based navigation
6. **Collapsible** - Auto-collapse long code
7. **Annotations** - Line-by-line explanations
8. **Editable** - Interactive code editing
9. **Combined** - Multiple features together
10. **Advanced** - Custom execution handlers

Each feature can be combined as needed to create the perfect code display for your documentation needs!
